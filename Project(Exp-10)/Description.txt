Goal:-----
--Design and implement a multi-party audio/video conferencing prototype using WebRTC for peer-to-peer media exchange and a Node.js + WebSocket signaling server for session coordination. The client is built in React, supporting small-group mesh calls with live controls, screen sharing, and adaptive quality management.

Core Components:-----
--Signaling Server (Node.js + WebSocket)
--Handles room create/join/leave.
--Relays SDP offers/answers and ICE candidates.
--Broadcasts participant presence and updates.
--Uses MongoDB for storing rooms, participants, and basic call logs.
--Supports JWT authentication and room-scoped tokens.
--WebRTC Peer Connections
--P2P mesh architecture (small rooms).
--Manages audio/video/screen tracks with dynamic add/remove and renegotiation.
--Auto ICE restarts and STUN/TURN for NAT traversal.
--Detects connectivity loss and reconnects gracefully.
--React Client
--Grid view of all participants with active-speaker highlighting.
--Mute/unmute mic, toggle camera, share screen.
--Device chooser for input/output.
--Connection quality indicators and adaptive bitrate/resolution.
--Automatic reconnection on network drops.
--Media & Quality Control
--Customizable getUserMedia constraints (HD, SD, Low).
--Simulcast or per-sender bitrate caps.
--Dynamic degradation preference under poor networks.
--Session Persistence (MongoDB)
--Stores room info, join/leave times, and call stats.
--Short-lived tokens for room access and participant sessions.
--Resilience & Security
--Auto reconnect with exponential backoff.
--Graceful teardown on tab close.
--JWT-based auth, CORS/origin checks, and optional end-to-end encryption (Insertable Streams).

Optional Enhancements:----
--In-room text chat and reactions using the same signaling channel.
--Structured logs, health endpoints, and basic metrics (participants, bitrate, packet loss).
--Local setup scripts for Node.js, MongoDB, React, and coturn.

Outcome:--
--A working small-group video conferencing prototype that demonstrates:
--Real-time WebRTC communication with adaptive behavior.
--Reliable NAT traversal and reconnection.
--Secure signaling and tokenized room access.
--Extendable foundation for full-scale conferencing apps.
